# תבנית Guru לתהליך חדש

> מומלץ שכל מדריך יהיה קצר, ממוקד משימה אחת, וקריא גם בזמן עבודה.

## מתי משתמשים במדריך הזה

- **מטרה:** מה התהליך פותר
- **למי זה מיועד:** מי מבצע
- **מתי להשתמש:** באיזה מצב פותחים את המדריך
- **זמן משוער:** כמה זמן זה לוקח

## לפני שמתחילים

- [ ] לוודא שיש את כל המידע הדרוש
- [ ] לוודא שיש גישה למערכות הרלוונטיות
- [ ] לוודא שיש אישור אם צריך

## מה צריך להכין

- פרטי עובד / משתמש
- מזהה מכשיר / נכס
- מספר קריאה / בקשה אם קיים
- מידע משלים רלוונטי
